DS360: Computer Networks

Midterm Review

Module 01

Terminologies:

a. Message:
the message is the information (data) to be communicated.
b. Sender:
The device that sends the message. It can be a computer, workstation, telephone handset, or television.
c. Receiver:
The device that receives the message.
d. Transmission Medium:
The physical path by which a message travels from sender to receiver.
e. Protocol:
a set of rules that govern data communication.

Communication between two devices can be:

Simplex Mode:
The communication is unidirectional, as in a one-way street. Only one of the two devices on link can transmit, the other can only receive.
Half-duplex Mode:
Each device can both send and receive, but not at the same time. When one device is sending, the other can only receive, and vise versa.
Full-duplex Mode:
Both stations can send and receive simultaneously.

Network Criteria

A network must be able to meet the following criteria:

  1. Performance
  2. Reliability
  3. Security

Physical Structures:

1. Types of Connection:
  • Point-to-Point: A dedicated link between two devices.
  • Multipoint: A connection in which more than two devices share a single link.
2. Topology: The physical representation of the relationship of all links and linking devices:
  • Mesh: Every device has a dedicated point-to-point link to every other device.
  • Star: Each device has a dedicated point-to-point link only to a central controller, usually called a hub.
  • Bus: one long cable acts as a backbone linking all devices in the network.
  • Ring: Each device has a dedicated point-to-point connection with only the two devices on either side of it.

Network Types:

  1. Local Area Network LAN
  2. Wide Area Network WAN

Switching:

a. Circuit-switched Network:
A dedicated connection, called a circuit, is always available between the two end systems.
b. Packet-switched Network:
The data transfers into a network in the form of packets. Each packet uses a different path toward the end system.

Module 02

1. Logical Connection:

means that we have a Layer-to-Layer communication

2. TCP/IP:

is a protocol suite. It is short for Transmission Control Protocol/Internet Protocol.

3. TCP/IP Layers are:

  1. Application: To allow access to network resources.
  2. Transport: To provide reliable process to process communications, and error recovery.
  3. Network: Move packets from source to destination. Provide internetworking.
  4. Data Link: To organize bits into frames. Provides hop-to-hop delivery.
  5. Physical: Transmits bits over a medium.

4. Encapsulation:

Data is moving from upper layer to lower layer while packing of data.

5. Decapsulation:

Data is moving from lower layer to upper layer while unpacking of data.

6. Multiplexing:

Means that a protocol can encapsulate a packet from several next-higher layer protocols.

7. Demultiplexing

Means that a protocol can decapsulate and deliver a packet to several next-higher layer protocols.

Modules 03

1. Transmission Impairment:

This means that the signal at the beginning of the medium is not the same as the signal at the end of the medium .

3. Impairment Causes:

a. Attenuation:
  1. It means loss of energy.
  2. When a signal, simple or composite, travels through a medium, it loses some its energy in overcoming the resistance of the medium.
  3. Amplifiers are used to Amplify the signal.
b. Distortion:
  • Means the signal changes its form and shape.
c. Noise:
  1. Thermal noise is the random motion of electrons in a wire, which creates an extra signal not originally sent by the transmitter.
  2. Induced noise comes from sources such as motors and appliances.
  3. Crosstalk is the effect of one wire on the other.
  4. Impulse noise is a spike (a signal with high energy in a very short period of time) that comes from power lines, lightning.

3. Performance:

a. Bandwidth:
The number of bits per second that a channel, a link, or even a network can transmit.
b. Throughput:
A measure of how much data we can send through a network.
c. Latency(Delay):
How long it takes for an entire message to completely arrive at the destination from the time the first bit is sent out from the source.
d. Jitter:
Different packets of data encounter different delays.

Module 04

Classes of Transmission Media

a. Guided:

i. Twisted Pair Cable:
  1. Two conductors (normally copper), each with its own plastic insulation, twisted together.
ii. Coaxial Cable:
  1. Carries signals of higher frequency than those in twisted pair cable.
  2. The outer metallic wrapping serves both as a shield against noise and as a second conductor.
  3. It has a much higher bandwidth
  4. Attenuations are much higher compared to twisted pair cable.
iii. Fiber-optic cable:
  1. Transmits signal in the form of light.
  2. Attenuations flatter than in the case of twisted-pair cable and coaxial cable.

b. Unguided:

i. Radio:
  1. Use omnidirectional antennas that send out signals in all directions.
  2. Useful for multicasting.
ii. Microwave
iii. Infrared:
  1. Used for short-range communication.

Module 05

1. Communication at the data-link layer is node-to-node.

2. Data-link Layer Services:

  1. Farming
  2. Flow Control
  3. Error Control
  4. Congestion Control

3. The data-link layer controls how the medium is used.

4. The data-link layer can be divided into two sub-layers:

  1. Data Link Control DLC
  2. Media Access Control MAC

Three Types of Addresses:

a. Unicasting means one-to-one communication:
The second digit in the Ethernet bits need to be an even number.
b. Multicasting means one-to-many communication:
The second digit in the Ethernet bits need to be an odd number.
c. Broadcasting means one-to-all communication:
All the digits in the Ethernet bits are 1s

6. ARP (Address Resolution Protocol)

Accepts an IP Address from the IP protocol, maps the address to the corresponding link-layer address, and passes it to the data-link layer.

7. Multiple-Access Protocol:

7.1. Random Access Protocols

i. Aloha
  1. There are potential collisions.
ii. CSMA/CD
  1. A station monitors the medium after it sends a frame to see if the transmission was successful or not.
iii. CSMA/CA
  1. Each station first listen to the medium (or check the state of the medium) before sending.
  2. Reduce possibility of collision.
  3. Collisions are avoided through the use of three strategies:
    1. Interframe space IFS
    2. Contention window.
    3. Acknowledgement.

7.2. Channelization Protocols:

i. Frequency-Division Multiple Access FDMA
  1. The available bandwidth is divided into frequency bands (channels)
  2. Each station is allocated a band to send its data.
ii. Time-Division Multiple Access TDMA
  1. Each station is allocated a time slot during which it can send data.

Module 06

1. Ethernet Evolution

a. Standard Ethernet 10 Mbps

1. Connectionless & Unreliable
2. Addressing:
  1. 6 bytes (48bits) address, normally written in hexadecimal notation, with a colon between bytes.
  2. The source is always a unicast address - the frame comes from only one station.
  3. The destination address can be unicast, multicast, or broadcast.
3. Access Method: CSMA/CD with 1-persistent method
4. Implementation:
  1. Coaxial cable (bus topology)
  2. A set of twisted-pair cables with a hub (star topology)
  3. Transmission is always broadcast.

b. Ethernet Frame Structure:

  1. Preamble
  2. SFD
  3. Dest. Address
  4. Source Address
  5. Type
  6. Data & Padding
  7. CRC

c. Fast Ethernet 100 Mbps

  1. The MAC sublayer was left unchanged.
  2. The transmission rate, access method, and implementation had to be changed.

d. Gigabit Ethernet 1 Gbps

e. 10 Gigabit Ethernet 10 Gbps

  1. Operates only in full-duplex mode.

Module 07

Discuss The Various Connecting Devices:

a. Hubs:
  1. Operates only in the physical layer.
  2. A repeater/hub receives a signal and, before it becomes too weak or corrupted, regenerates and retimes the original bit pattern.
b. Switches:
  1. Operate in both the physical and the data-link layers.
  2. Regenerates the signal it receives.
c. Routers:
  1. A three-layer device; it operates in the physical, data-link, and network layers.
  2. Can connect the whole system to the internet.

The process for finding the spanning tree involves three steps:

  • Each switch broadcasts its ID so that all switches know which one has the smallest ID.
  • The algorithm tries to find the shortest path from the root switch to every other switch or LAN.
  • We mark the ports the are part of the path. We also mark ports that are not part of the path.
